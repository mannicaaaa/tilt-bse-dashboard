# Tilt frontend

React + Tailwind dashboard generated by [Claude Design](https://claude.ai/design). No build step — opens directly as `index.html` using React UMD + Babel-in-browser + Tailwind CDN.

## Files

| File | Role |
|---|---|
| `index.html` | Entry. Loads React, Babel, Tailwind, scripts. |
| `api.js` | **API client** — fetch wrapper for the 11 FastAPI endpoints + adapters mapping API shape → `TiltData` shape. |
| `data.jsx` | Seed data (`window.TiltData`). Used when `TILT_API_BASE` is not set, so the dashboard is always demoable. |
| `icons.jsx` / `primitives.jsx` / `chrome.jsx` | Design system. |
| `screen-*.jsx` | One file per screen: dashboard, portfolio, scan, stock, backtest. |
| `app.jsx` | Routing + state. |
| `tweaks-panel.jsx` | Live-tunable design tokens (Claude Design artifact — leave for the polished look). |

## Mock vs live data

The dashboard is **always demoable**. By default it renders against the hand-tuned mock data in `data.jsx` — no backend required. To switch to live FastAPI data, uncomment the API-base line in `index.html`:

```html
<script>window.TILT_API_BASE = "https://tilt-api.onrender.com";</script>
```

Or set it dynamically in dev tools:

```js
window.TILT_API_BASE = "http://localhost:8000";
location.reload();
```

`api.js` exposes:

- `TiltAPI.isLive()` → boolean
- `TiltAPI.{health, refresh, scanRally, scanRallyConviction, scanRallyBySector, scanAveraging, scanTraps, sectorsHeatmap, portfolio, backtestRally, stock}` — one method per endpoint, all return Promises.
- `TiltAPI.fetchTiltData()` — fan out the 3 endpoints the dashboard needs and assemble a `TiltData`-shaped object.
- `TiltAPI.adapters.{adaptSector, adaptScanResult, adaptHolding}` — pure shape converters; useful for testing the live wiring.

## Running locally

```bash
# Backend
cd ..
.venv/bin/uvicorn tilt.api.app:app --reload --port 8000

# Frontend (any static server)
cd frontend
python -m http.server 5173
# open http://localhost:5173 — runs in mock mode unless you set TILT_API_BASE
```

For local live-mode, paste in dev tools after page load:

```js
window.TILT_API_BASE = "http://localhost:8000";
location.reload();
```

## Deploying to Vercel

`frontend/` is a static directory — no build step. Vercel auto-deploys from `git push`. The root project is the Python API; this directory deploys as a separate Vercel project.

Set `TILT_API_BASE` in `index.html` to the Render API URL before pushing.

## Why not a proper React build pipeline?

Claude Design emitted UMD-React + babel-in-browser deliberately — it's an interview-friendly artifact (the interviewer can `view-source` and read the code, no `dist/` opacity). A `vite build` step is a fine evolution if the project grows past prototype.
